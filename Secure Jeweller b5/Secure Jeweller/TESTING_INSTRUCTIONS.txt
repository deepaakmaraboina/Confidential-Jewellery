6. SYSTEM ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Backend (Python):
  main.py
  ├─ /api/gold-price → Returns current price as JSON
  ├─ /api/gold-price/refresh → Force refresh from APIs
  └─ WebSocket handlers:
     ├─ connect → Send price to new clients
     ├─ disconnect → Clean up
     ├─ request_gold_price → Return price on demand
     └─ [Background] emit_gold_price_update every 30s

gold_price_service.py
  ├─ get_gold_price() → Fetch with cache (60s timeout)
  ├─ get_gold_price_per_gram() → Price per gram
  └─ API sources: openexchangerates, exchangerate_api, fixer

Frontend (JavaScript in base.html):
  ├─ Connect to WebSocket on page load
  ├─ Listen for 'gold_price_update' events
  ├─ Update .gold-price-value elements
  ├─ Fallback: Fetch /api/gold-price every 30s
  └─ Show timestamps with updates

Templates:
  ├─ admin/dashboard.html → Gold price card
  ├─ customer/dashboard.html → Gold price card
  └─ owner/dashboard.html → Gold price card

Styling (style.css):
  ├─ .gold-price-value.loading → Pulse animation
  └─ Responsive card layout
